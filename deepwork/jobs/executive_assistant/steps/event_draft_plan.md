# Draft Event Plan

## Objective

Create a comprehensive event plan that selects the best options, builds an agenda, and prepares draft invitations — ready for the user to review and approve.

## Task

Synthesize the requirements and researched options into a cohesive event plan. Make clear recommendations (not just options) while noting alternatives. Include an actionable timeline and draft invitation text.

### Process

1. **Review inputs**
   - Read event_requirements.md for constraints and preferences
   - Read event_options.md for researched venues, catering, and logistics

2. **Make recommendations**
   - Select the best venue option with rationale
   - Select catering approach with rationale
   - Propose logistics plan (transportation, accommodation, equipment)

3. **Build the agenda**
   - Create a time-blocked agenda for the event
   - Include setup and teardown time
   - Account for meals, breaks, and transitions

4. **Draft invitation text**
   - Write invitation email/message text appropriate for the event formality
   - Include: what, when, where, RSVP instructions, dress code, logistics info
   - Keep tone appropriate to the event type

5. **Create execution timeline**
   - List all tasks needed before the event with deadlines
   - Include booking deadlines, payment milestones, communication touchpoints

## Output Format

### event_plan.md

The complete draft event plan.

**Structure**:

```markdown

- **Selected**: [caterer/approach]
- **Menu Concept**: [overview]

---

[Full draft invitation text, ready to customize and send]

---

## Execution Timeline

| Deadline | Task   | Owner |
```

## Quality Criteria

- The plan covers venue, agenda, catering, and logistics
- A clear timeline of tasks and deadlines is present
- Draft invitation text is included and ready to customize
- Budget estimate is within the stated range
- Recommendations have clear rationale, not just listings
- Open questions are flagged rather than silently assumed

## Context

This is the main deliverable of the event planning workflow. It should be comprehensive enough that the user can approve it with minimal changes, or provide targeted feedback for the finalization step.
